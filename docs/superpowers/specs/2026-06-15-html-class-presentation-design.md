- 阶段：内容与布局设计
- 状态：已确认，可作为后续 HTML PPT 实现依据

## 1. 目标与边界

### 1.1 汇报目标

本次汇报用于课堂展示，时长控制在 5 分钟以内。PPT 采用 HTML 形式实现，但本设计文档只定义内容、布局、讲述逻辑和素材使用规则，不定义最终视觉风格。

本次汇报要让老师和同学记住两个核心印象：

1. 项目的模型训练部分是真实完成的，不只是界面展示。
2. 项目已经形成可用闭环，不只是单张结果图。

### 1.2 表达方向

整体方向采用：

- 课堂答辩型骨架
- 视觉展示型表达

也就是说，信息组织要稳定清晰，便于课堂理解；页面呈现可以有展示感，但不能为了视觉效果牺牲理解速度。

### 1.3 展示边界

本阶段只做以下内容：

- 规划 PPT 叙事结构
- 规划逐页内容与布局
- 规定术语通俗化规则
- 指定现有素材的使用位置
- 规划演讲节奏和演示插入点

本阶段不做以下内容：

- 不实现 HTML 页面
- 不确定最终配色、字体、动画风格
- 不自动提交 git
- 不扩展新的系统功能

## 2. 受众与表达原则

### 2.1 受众

主要受众为课堂教师和同学，不假设所有人都熟悉医学图像分割、Attention UNet 或分割评估指标。

### 2.2 总体表达原则

整场汇报遵循以下原则：

1. 每页只传达一个核心判断。
2. 先让观众看懂，再让观众觉得专业。
3. 专业术语可以保留，但必须立刻翻译成通俗表达。
4. 不堆砌技术名词，不照着页面朗读。
5. 优先展示最能证明“模型真实训练过”和“系统真实可用”的内容。

## 3. 术语通俗化规则

### 3.1 规则

1. 先说中文含义，再说术语。
2. 同一个术语只承担一个信息点。
3. 缩写第一次出现必须解释。
4. 指标只解释“它代表什么”，不展开公式推导。
5. 模型部分重点解释“为什么这样做”，不深入到底层数学细节。
6. 3D 展示部分重点解释“帮助观察什么”，不强调实现术语。

### 3.2 示例

- “Dice 指标，也就是预测区域和真实标注重合得有多好。”
- “Attention UNet 可以理解为更关注真正有病灶的区域，减少背景干扰。”
- “四类病灶分别是出血、渗出、微动脉瘤和棉絮斑。”

### 3.3 页面约束

- 第 2 页只讲系统做什么，不堆术语。
- 第 4 页术语最多，但每个术语后面都要有白话解释。
- 第 5 页重点讲结果含义，不念指标数字。
- 第 6 页全部回到通俗总结，不再引入新术语。

## 4. 汇报结构总览

本次 PPT 采用以下叙事顺序：

1. 案例开场
2. 系统闭环
3. 系统展示与短演示
4. 回到模型方法
5. 展示训练结果与分割效果
6. 总结项目价值
7. Q&A 收尾

这是一个“先看成品，再解释模型”的结构，适合课堂展示，也更能抓住注意力。

## 6. 全局版式规范

本次 PPT 的版式只规定结构，不规定具体风格。

### 6.1 通用规范

- 画幅使用 16:9
- 四周保留安全边距，尤其是底部
- 每页只保留一个主标题和一个主视觉中心
- 每页信息层级不超过 2 级
- 每页正文控制在 3 到 5 条以内
- 不做满屏小字和密集表格

### 6.2 逐页版式骨架

- 第 1 页：大图主导，文字置角
- 第 2 页：中轴流程页
- 第 3 页：一主图加多辅图
- 第 4 页：左图右文
- 第 5 页：上指标，下效果
- 第 6 页：三段式总结
- 第 7 页：居中收束

## 7. 素材使用规则

### 7.1 主素材原则

1. 一个页面只设置一个主图。
2. 同类截图不重复多次出现。
3. 辅助图只负责补充，不与主图争抢注意力。
4. 不能让观众在不听解释的情况下看不懂页面结构。

### 7.2 删减规则

以下内容不进入主画面：

- API 清单
- 数据库表结构
- 训练日志原始 JSON
- 阈值 JSON 原文件
- 复杂损失函数公式推导
- 网络层参数细节
- 过多后台页面细节

### 7.3 课堂阅读优先级

素材选择优先级如下：

1. 一眼能说明项目成果的页面
2. 能证明模型真实训练过的图
3. 能证明系统闭环真实存在的截图
4. 只在讲稿层使用、但不必放上屏幕的工程细节

## 8. 演讲节奏规划

### 8.1 总体节奏

全场目标时长控制在 4 分 30 秒到 5 分钟。

推荐分配如下：

- 第 1 页：25 秒
- 第 2 页：35 秒
- 第 3 页：60 秒
- 第 4 页：50 秒
- 第 5 页：50 秒
- 第 6 页：40 秒
- 第 7 页：10 秒

### 8.2 讲述原则

- 每页只讲一个判断，不逐字朗读
- 开场 30 秒内进入案例
- 演示严格控制在 30 到 60 秒
- 模型页和结果页讲人话，不堆术语
- 结束前主动把项目重新概括成一句完整的话

### 8.3 总控话术

整场汇报的主线可以概括为：

“我不是只训练了一个分割模型，也不是只做了一个界面，而是把病灶分割、结果分析和系统展示做成了一个完整闭环。”

## 9. 后续实现约束

当进入 HTML PPT 实现阶段时，必须遵守以下约束：

1. 本文档优先级高于临时页面想法。
2. 若实现时需要调整风格，不得破坏已确认的内容顺序和版式主次。
3. 若新增页面，必须先说明为什么 5 分钟内仍然讲得完。
4. 若删减页面，不能同时削弱“模型真实训练过”和“系统真实可用”这两个核心印象。

## 10. 最终验收标准

一份合格的 HTML 课堂汇报 PPT，应满足以下标准：

1. 老师在 5 分钟内能听懂项目做了什么。
2. 老师能明确感受到模型训练不是摆设。
3. 老师能明确感受到系统是完整闭环，而不是单页面演示。
4. 全程术语不过载，非本方向同学也能跟住。
5. 页面布局服务于讲述，而不是反过来拖慢讲述。
