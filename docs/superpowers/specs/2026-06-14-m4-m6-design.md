# M4-M6 设计说明

## 范围

本阶段完成后端推理调度、前端核心页面，以及上传到报告展示的联调闭环。

## M4 后端

- 新增 `ImageService`：校验 jpg/jpeg/png/bmp/webp，保存上传原图。
- 新增 `InferenceService`：支持 `MODEL_BACKEND=auto|real|placeholder`。
- 新增 `POST /api/diagnose`：接收 `image` 与 `patient_id`，保存原图，执行推理，写入 `diagnoses` 表并返回诊断结果。
- 新增 `GET /api/images/<path>`：安全返回原图或轮廓图。
- 新增基础统计接口：`/api/stats/overview`、`/api/stats/lesions`、`/api/stats/trend`。

## M5 前端

- 搭建 Vite + React + TypeScript + Ant Design + React Router。
- 建立 axios 请求层与 TypeScript 类型。
- 实现核心页面：
  - Dashboard
  - Diagnose
  - DiagnosisDetail
  - Patients
  - PatientDetail
- 使用毛玻璃主题，保持文档定义的视觉方向。
- Diagnose 页面支持患者选择、上传图片、发起诊断、展示量化报告和轮廓图。

## M6 联调

- Vite 代理 `/api` 到 Flask `5000`。
- 验证上传图片到后端推理，再回到前端展示报告。
- 覆盖 placeholder 和 auto/real 的后端模式边界。

## 暂不包含

- Three.js 真实 3D 球体病灶分布，留到 M7。
- ECharts 3D 图表深度实现，留到 M8。
- 用户登录、分类任务、移动端、Docker。
