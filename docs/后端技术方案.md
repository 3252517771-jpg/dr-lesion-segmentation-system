- services/ 直接返回 Response 对象 — 只返回数据，由 api/ 包装

## 四、推理管线详解

### 4.1 推理调度逻辑（inference_service.py）

```
POST /api/diagnose
    ↓
InferenceService.run_diagnosis(image_path)
    ↓
检查 MODEL_BACKEND 环境变量
    ├── "placeholder" → PlaceholderInference.run()
    │   └── 返回假数据（面积/数量/轮廓图）
    ├── "real" → RealInference.run()
    │   ├── 加载 attention_unet_dr.pth
    │   ├── 加载失败 → 直接报错
    │   └── 成功 → 推理 → 返回结果
    └── "auto" → RealInference.run() 优先
        ├── 模型加载成功 → 推理 → 返回结果
        └── 加载失败 → 自动回退到 PlaceholderInference.run()
    ↓
统一返回格式：
{
    "lesion_areas": {"HE": 3.2, "EX": 1.8, "MA": 0.6, "SE": 0.9},
    "lesion_counts": {"HE": 15, "EX": 7, "MA": 28, "SE": 2},
    "contour_path": "/uploads/contours/xxx.png",
    "severity": "中度 NPDR"
}
```

### 4.2 推理流程（inference.py）

```
predict(image_path) → dict
    ├── load_image(image_path)
    │   └── PIL 打开 → RGB → resize 512×512
    ├── preprocess(image)
    │   └── ToTensor → Normalize (ImageNet mean/std) → unsqueeze batch
    ├── model.forward(input_tensor)
    │   └── → output: [1, 4, 512, 512] (logits)
    ├── sigmoid → threshold 0.5 → binary masks [1, 4, H, W]
    ├── restore_to_original_size(masks, orig_size)
    │   └── 双线性插值恢复到原图尺寸
    ├── compute_lesion_areas(masks, image_size)
    │   └── 各病灶像素数 / 图像总像素 → 面积占比
    ├── compute_lesion_counts(mask)
    │   └── 连通域分析（cv2.connectedComponents）→ 病灶数量
    └── generate_contour_image(original, masks)
        └── 在原图上画彩色轮廓线（cv2.drawContours）→ 保存 PNG
```

### 4.3 病灶面积占比计算规则

```python
# 面积占比 = 该病灶前景像素数 / 图像总像素数
# 即：病灶在整张眼底图中所占的面积百分比
# HE: red, EX: yellow, MA: green, SE: blue
LESION_COLORS = {
    "HE": (255, 0, 0),      # 红色 - 出血
    "EX": (255, 255, 0),    # 黄色 - 渗出
    "MA": (0, 255, 0),      # 绿色 - 微动脉瘤
    "SE": (0, 0, 255),      # 蓝色 - 棉絮斑
}
```

### 4.4 综合评估（severity）规则

基于病灶面积占比的经验阈值（非临床标准）：

| Severity | 条件 |
|----------|------|
| 轻度 NPDR | 仅 MA 检出，或各病灶面积占比均 < 1% |
| 中度 NPDR | HE 或 EX 面积占比 ≥ 1% 且 < 5% |
| 重度 NPDR | HE 或 EX 面积占比 ≥ 5%，或 SE ≥ 3% |
| 正常 | 所有病灶检出面积为 0 |

## 六、异常处理统一规范

### 6.1 错误响应格式

```json
{
  "error": {
    "code": "PATIENT_NOT_FOUND",
    "message": "患者不存在"
  }
}
```

### 6.2 HTTP 状态码使用规范

| 状态码 | 触发条件 | 消息模板 |
|--------|---------|---------|
| 200 | 成功 | — |
| 400 | 参数校验失败 / 无上传图片 | "图片格式不支持" / "患者姓名不能为空" |
| 404 | 资源不存在 | "患者不存在" / "诊断记录不存在" |
| 413 | 图片超 10MB | "图片大小不能超过 10MB" |
| 415 | 图片格式不支持 | "仅支持 jpg/png/bmp/webp 格式" |
| 422 | 参数格式错误 | "日期格式错误" |
| 500 | 推理失败 / 服务器错误 | "模型加载失败" / "服务器内部错误" |

### 6.3 异常类型定义

```python
# backend/errors.py

class AppError(Exception):
    def __init__(self, code: str, message: str, status: int = 400):
        self.code = code
        self.message = message
        self.status = status

class PatientNotFoundError(AppError):
    def __init__(self):
        super().__init__("PATIENT_NOT_FOUND", "患者不存在", 404)

class DiagnosisNotFoundError(AppError):
    def __init__(self):
        super().__init__("DIAGNOSIS_NOT_FOUND", "诊断记录不存在", 404)

class InvalidImageError(AppError):
    def __init__(self, msg="图片格式不支持"):
        super().__init__("INVALID_IMAGE", msg, 400)

class ModelLoadError(AppError):
    def __init__(self):
        super().__init__("MODEL_LOAD_FAILED", "模型加载失败", 500)

class InferenceError(AppError):
    def __init__(self, msg="推理失败"):
        super().__init__("INFERENCE_FAILED", msg, 500)
```

## 七、推理模式切换

### 7.1 环境变量控制

```bash
# .env
MODEL_BACKEND=auto       # auto | real | placeholder
UPLOAD_FOLDER=uploads/originals
CONTOUR_FOLDER=uploads/contours
MAX_IMAGE_SIZE=10485760  # 10MB
ALLOWED_EXTENSIONS=jpg,jpeg,png,bmp,webp
```

### 7.2 三种模式行为

| 模式 | 无模型文件 | 有模型文件 |
|------|-----------|-----------|
| `placeholder` | 返回假数据 | 返回假数据（忽略模型） |
| `real` | 报错 500 | 正常推理 |
| `auto` | 回退 placeholder | 正常推理 |

### 7.3 健康检查响应

```json
{

  "db": true,
  "model_backend": "auto",
  "model_loaded": true,
  "model": "AttentionUNet",
  "segmentation_classes": ["HE", "EX", "MA", "SE"]
```
