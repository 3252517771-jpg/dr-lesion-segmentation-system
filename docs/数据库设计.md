## 三、各对象业务场景

### 3.1 Patient（患者）

| 场景 | 操作 | 涉及字段 | 触发时机 |
|------|------|---------|---------|
| 创建患者 | INSERT | name, gender, age, patient_id | 用户填写患者信息提交 |
| 查看患者 | SELECT | 全部 | 患者列表页加载 |
| 搜索患者 | SELECT + LIKE | patient_id, name | 搜索框输入 |
| 更新患者 | UPDATE | name, gender, age | 编辑患者信息 |
| 删除患者 | UPDATE is_deleted | is_deleted | 用户删除患者 |

### 3.2 Diagnosis（诊断记录）

| 场景 | 操作 | 涉及字段 | 触发时机 |
|------|------|---------|---------|
| 上传诊断 | INSERT | patient_id, image_path, contour_path, lesion_areas (JSON), lesion_counts (JSON), severity | 用户上传图片 + 推理完成 |
| 查看诊断详情 | SELECT by id | 全部 | 进入诊断详情页 |
| 查看患者诊断记录 | SELECT by patient_id | 全部 | 进入患者详情页 |
| 查看所有诊断列表 | SELECT | 全部 | Dashboard 最近诊断 |
| 更新备注 | UPDATE | notes | 医生编辑备注 |
| 删除诊断（级联患者删除） | UPDATE is_deleted | is_deleted | 患者被删除时 |

## 四、数据库表结构

### 4.1 命名规范

| 规则 | 示例 |
|------|------|
| 表名：小写下划线复数 | `diagnoses` |
| 字段名：小写下划线 | `patient_id`, `contour_path` |
| 主键：`id` Integer 自增 | `id INTEGER PRIMARY KEY AUTOINCREMENT` |
| 外键：`{表名}_id` | `patient_id INTEGER REFERENCES patients(id)` |
| 时间戳：`created_at` | 自动填充 |
| 软删除：`is_deleted` | Boolean 字段 |

### 4.2 所有表总览

| # | 表名 | 说明 | v1 必需 |
|---|------|------|---------|
| 1 | `patients` | 患者信息 | ✅ |
| 2 | `diagnoses` | 诊断记录 | ✅ |

### 4.3 建表语句

```sql
-- ============================================================
-- 1. patients 患者信息
-- ============================================================
CREATE TABLE patients (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    name            VARCHAR(100) NOT NULL,           -- 患者姓名
    gender          VARCHAR(10) NOT NULL,            -- 性别：男/女
    age             INTEGER NOT NULL CHECK (age >= 0 AND age <= 150),  -- 年龄
    patient_id      VARCHAR(50) NOT NULL UNIQUE,     -- 病历号（唯一）
    created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    is_deleted      BOOLEAN NOT NULL DEFAULT 0       -- 软删除标记
);

CREATE INDEX idx_patients_patient_id ON patients(patient_id);
CREATE INDEX idx_patients_name ON patients(name);
CREATE INDEX idx_patients_deleted ON patients(is_deleted);

-- ============================================================
-- 2. diagnoses 诊断记录
-- ============================================================
CREATE TABLE diagnoses (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    patient_id      INTEGER NOT NULL REFERENCES patients(id) ON DELETE CASCADE,
    image_path      VARCHAR(500) NOT NULL,           -- 原图相对路径
    contour_path    VARCHAR(500),                    -- 病灶轮廓图相对路径（可为空，placeholder 模式可能无图）
    lesion_areas    TEXT NOT NULL DEFAULT '{}',       -- JSON: {"HE": 3.2, "EX": 1.8, "MA": 0.6, "SE": 0.9}
    lesion_counts   TEXT NOT NULL DEFAULT '{}',       -- JSON: {"HE": 15, "EX": 7, "MA": 28, "SE": 2}
    severity        VARCHAR(50) NOT NULL DEFAULT '正常',  -- 正常/轻度 NPDR/中度 NPDR/重度 NPDR
    notes           TEXT DEFAULT '',                 -- 医生备注
    created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    is_deleted      BOOLEAN NOT NULL DEFAULT 0       -- 软删除标记
);

CREATE INDEX idx_diagnoses_patient ON diagnoses(patient_id);
CREATE INDEX idx_diagnoses_created ON diagnoses(created_at DESC);
CREATE INDEX idx_diagnoses_deleted ON diagnoses(is_deleted);

-- 联合索引：按患者查询未删除的诊断记录
CREATE INDEX idx_diagnoses_patient_active ON diagnoses(patient_id, is_deleted);
```

| datetime | DATETIME | 时间戳 |
| bool | BOOLEAN (INTEGER 0/1) | 布尔值 |

### 5.1 JSON 字段处理

由于 SQLite 无原生 JSONB，`lesion_areas` 和 `lesion_counts` 以 TEXT 存储 JSON 字符串。

**Python 侧自动序列化/反序列化：**

```python
# database/models.py
import json
from sqlalchemy import TypeDecorator, Text

class JSONField(TypeDecorator):
    """SQLite 兼容的 JSON 字段"""
    impl = Text

    def process_bind_param(self, value, dialect):
        if value is None:
            return "{}"
        return json.dumps(value)

    def process_result_value(self, value, dialect):
        if value is None:
            return {}
        return json.loads(value)
```

## 六、数据库模型定义（SQLAlchemy）

```python
# backend/database/models.py
from datetime import datetime
from extensions import db

class Patient(db.Model):
    __tablename__ = "patients"

    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(100), nullable=False)
    gender = db.Column(db.String(10), nullable=False)
    age = db.Column(db.Integer, nullable=False)
    patient_id = db.Column(db.String(50), unique=True, nullable=False)
    created_at = db.Column(db.DateTime, default=datetime.utcnow)
    updated_at = db.Column(db.DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
    is_deleted = db.Column(db.Boolean, default=False)

    diagnoses = db.relationship("Diagnosis", backref="patient", lazy="dynamic")

    def to_dict(self):
        return {
            "id": self.id,
            "name": self.name,
            "gender": self.gender,
            "age": self.age,
            "patient_id": self.patient_id,
            "diagnosis_count": self.diagnoses.filter_by(is_deleted=False).count(),
            "created_at": self.created_at.isoformat() if self.created_at else None,
        }

    def soft_delete(self):
        self.is_deleted = True
        # 级联软删除诊断记录
        for diagnosis in self.diagnoses:
            diagnosis.soft_delete()


class Diagnosis(db.Model):
    __tablename__ = "diagnoses"

    id = db.Column(db.Integer, primary_key=True)
    patient_id = db.Column(db.Integer, db.ForeignKey("patients.id"), nullable=False)
    image_path = db.Column(db.String(500), nullable=False)
    contour_path = db.Column(db.String(500), nullable=True)
    lesion_areas = db.Column(JSONField, default=dict)
    lesion_counts = db.Column(JSONField, default=dict)
    severity = db.Column(db.String(50), default="正常")
    notes = db.Column(db.Text, default="")
    created_at = db.Column(db.DateTime, default=datetime.utcnow)
    updated_at = db.Column(db.DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
    is_deleted = db.Column(db.Boolean, default=False)

    def to_dict(self):
        return {
            "id": self.id,
            "patient_id": self.patient_id,
            "patient_name": self.patient.name if self.patient else None,
            "image_path": self.image_path,
            "contour_path": self.contour_path,
            "lesion_areas": self.lesion_areas,
            "lesion_counts": self.lesion_counts,
            "severity": self.severity,
            "notes": self.notes,
            "created_at": self.created_at.isoformat() if self.created_at else None,
        }

    def soft_delete(self):
        self.is_deleted = True
```

## 八、数据流全景

```
用户上传眼底图
    ↓
前端 POST /api/diagnose (multipart: image + patient_id)
    ↓
后端 ImageService 校验（格式/大小）
    ↓
InferenceService 推理
    ├── ML 层：Attention UNet 前向推理
    ├── 后处理：病灶面积计算 + 数量统计
    └── 生成轮廓图保存
    ↓
写入 diagnoses 表 (INSERT)
    ↓
返回 JSON 诊断结果
    ↓
前端渲染量化报告 + 轮廓图
    ↓
用户切换 Tab → Three.js 3D 球体展示
```

## 九、同步更新说明

### 9.1 前端类型与数据库的对应关系

```
后台表 (SQLite)
    ↓ SQLAlchemy ORM

    ↓ JSON API Response

    ↓ React State

```

### 9.2 新增字段注意事项

- 新增诊断相关字段时，必须同步更新：
  1. `database/models.py` — Diagnosis 模型
  2. `src/types/diagnosis.ts` — 前端类型
  3. API 响应中的 to_dict() 方法
  4. 前端页面对应的展示组件
- 病灶颜色映射 `LESION_COLORS` 前后端代码中必须保持一致
- 病灶类别顺序 `LESION_ORDER` 是固定的 `["HE", "EX", "MA", "SE"]`，前/后端/3D/图表统一使用此顺序
